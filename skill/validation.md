# Submodule: Bridge Output Quality Gate

This document is a final-pass checklist for code generated by the Solana Bridge Skill. Load it after the task-specific submodule and before finalizing an answer.

## Core Review Rules

1. **No Phantom Context**: Confirm every account, PDA seed, instruction argument, discriminator, and account field came from the user's IDL, source code, or explicit prompt.
2. **Modern RPC Boundary**: Direct Solana RPC code must use Web3.js v2 primitives. Anchor-specific React hooks may use the aliased `@solana/web3.js-v1` bridge only when required by Anchor.
3. **Strict TypeScript**: Avoid `any` except where Anchor's generated dynamic account namespace forces it. Prefer named interfaces, exported result types, and `unknown` for caught errors.
4. **Browser Compatibility**: Frontend snippets must not require Node-only globals. If `Buffer` is unavoidable, explicitly state the required polyfill.
5. **Cleanup and Cancellation**: Every subscription helper or React effect must expose teardown through `AbortController`, unsubscribe callbacks, or effect cleanup.
6. **User-Facing Errors**: Generated transaction hooks must expose readable errors and parse Anchor custom errors when the IDL provides an `errors` array.
7. **Dependency Clarity**: Final answers must state the package imports needed by the generated code.
8. **No Placeholders in Executable Paths**: Do not leave fake signatures, fake addresses, mock RPC responses, or incomplete instruction account lists in generated production code.

## Final Answer Checklist

Before replying to the user, verify that the answer includes:

- The generated files or functions and where they belong.
- Required dependencies and aliases.
- Any assumptions about IDL shape, account order, PDA seeds, or Token-2022 extension layout.
- A note about what was intentionally not generated because required source data was missing.
- A concise usage example only if it helps the user integrate the code.

## Common Failure Patterns To Catch

- Importing `PublicKey`, `Connection`, or `Transaction` from `@solana/web3.js` v2 in frontend bridge code.
- Passing an `Address` directly into Anchor `.accounts()` where Anchor expects a v1 `PublicKey`.
- Filtering Anchor accounts at offset `0` instead of skipping the 8-byte discriminator.
- Mapping Anchor `u64` or `i64` to `number`.
- Mapping `{ "option": T }` inconsistently as both `T | null` and an optional property.
- Forgetting that Token-2022 extensions start after the base account layout plus account type byte.
- Swallowing subscription errors without surfacing useful diagnostics during development.
